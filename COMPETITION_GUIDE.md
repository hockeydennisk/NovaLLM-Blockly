# 競賽演示指南

## 評審簡報架構（3-5分鐘）

### 開場：問題陳述（30秒）

**現況痛點**：
- 每次使用 NovaLLM 都要重新想 Prompt
- 好用的提示詞無法有效分享與重用
- 非技術人員不知道如何下好的 Prompt
- 優質 Prompt 無法成為企業知識資產

**一句話解決方案**：
> 我們把 Prompt 變成積木，讓提示詞工程變成人人都會的視覺化操作。

---

### 產品展示（2分鐘）

#### Demo 情境：QA 工程師分析測試 Log

**Step 1: 設定角色** (15秒)
- 拖曳「Persona 積木」到工作區
- 設定角色：資深 QA 工程師
- 設定專長：測試分析、缺陷追蹤
- 選擇語氣：專業嚴謹

**Step 2: 定義任務** (15秒)
- 拖曳「Task 積木」
- 輸入任務：分析以下測試 log 並找出根本原因

**Step 3: 提供情境** (15秒)
- 拖曳「Context 積木」
- 貼上測試 log 資料

**Step 4: 設定限制** (15秒)
- 拖曳「Constraint 積木」
- 勾選：使用條列式
- 設定：字數不超過 300 字
- 語言：繁體中文

**Step 5: 指定格式** (10秒)
- 拖曳「Output Format 積木」
- 選擇：標題+摘要+建議

**Step 6: 執行與儲存** (30秒)
- 右側即時顯示組合後的 Prompt
- 點擊「執行」→ 展示 NovaLLM 回應
- 點擊「儲存」→ 命名為「QA Log 分析模板」
- 點擊「分享」→ 展示 XML 分享功能

**亮點強調**：
- 下次遇到相同情境，直接「載入」模板即可
- 團隊成員可以互相分享優質模板
- ADM 也能使用，不需要會寫 Prompt

## 評審可能的提問與回答

### Q1: 這跟現有的 Prompt 管理工具有什麼不同？

**A**: 現有工具多是「文字輸入框+儲存功能」，我們的創新在於：
1. **視覺化**：用積木而非文字，降低學習門檻
2. **模組化**：每個積木是獨立的 Prompt 片段，可重組
3. **語意化**：積木本身就說明了用途（角色、任務、限制等）
4. **教育性**：使用過程就是在學習如何寫好 Prompt

### Q2: 技術可行性如何？多久能完成？

**A**:
- **MVP 已完成**：基於 Blockly + React，2-3天即可完成基礎版本
- **NovaLLM 整合**：只需 API 串接，1天內完成
- **Supabase 資料庫**：已設計好 schema，支援分享與版本控管
- **現在就能用**：Demo 環境已可運作

### Q3: 如何確保分享的 Prompt 品質？

**A**:
1. **回饋機制**：每個 Prompt 可按「好用」或「不準確」
2. **熱門排行**：顯示使用次數最多、評價最高的模板
3. **官方推薦**：由內部專家審核推薦優質模板
4. **版本追蹤**：記錄每次修改，可回溯優化過程

### Q4: 非技術人員真的會用嗎？

**A**:
- **降低認知負擔**：拖曳積木比寫文字簡單
- **預設選項**：每個積木都有合理的預設值
- **即時預覽**：右側立刻看到組合結果
- **ADM 友善**：專為非技術人員設計的語意化介面

### Q5: 跟 Scratch 的角色概念有何不同？

**A**:
- **Scratch 的角色**：畫面上的視覺角色（貓、狗等）
- **我們的 Persona**：AI 的人格與專業（工程師、分析師等）
- **適用場景**：企業工作流程，而非教育遊戲
- **進階用法**：可在同一流程中切換 Persona（QA 分析 → PM 摘要）

### Q6: 如何實現任務串連？

**A**:
1. **輸出孔設計**：每個任務積木右側有輸出孔
2. **輸入孔接收**：下一個任務可連接前一個的輸出
3. **資料選擇**：可選擇使用完整回覆或特定欄位
4. **範例流程**：
   - 任務A：分析資料 → 輸出分析結果
   - 任務B：根據分析結果產生報告 → 輸出報告
   - 任務C：將報告翻譯成英文
